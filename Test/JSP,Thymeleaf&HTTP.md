### SSR과 CSR의 차이점
**SSR(JSP,Thymeleaf)**은 서버에서 완성된 HTML을 만들어 보내주어 초기 로딩이 빠르고 검색 엔진 최적화에 유리함<br>
**CSR(React)**은 브라우저에서 JS로 화면을 그려 초기 로딩은 느리지만 이후 UX가 부드러움

### JSP와 Thymelaef의 가장 큰 차이점은?
JSP는 서블릿으로 컴파일되어 실행되는 반면,<br>
Thymeleaf는 순수 HTML 구조를 유지하는 템플릿 엔진이라 퍼블리셔와 협업이 좋음

### MVC 패턴에서 View의 역할
컨트롤러로부터 전달받은 데이터(Model)를 사용자에게 보여주는 UI화면 역할 담당

### GET과 POST 차이점
GET은 데이터 조회용으로 URL에 파라미터가 노출되고 캐싱이 됨<br>
POST는 데이터 생성용으로 Body에 데이터를 담아 보내며 멱등성이 없고 캐싱이 되지 않음

**멱등성** : 동일한 요청을 여러 번 보내더라도 서버의 상태가 처음 한 번 보냈을 때와 똑같이 유지되는 성질

### HTTTP 상태 코드

* 200 성공
* 400 클라이언트 요청 오류
* 401 로그인 필요
* 403 권한 없음
* 404 리소스 없음
* 500 서버 내부 오류

### 쿠키와 세션의 차이점
쿠키는 브라우저에 저장, 세션은 서버 메모리에 저장<br>
보안이 종요한 정보는 세션에 저장해야함

### RESTful API란 무엇이며 설계 원칙은?

HTTP URI로 자원을 명시하고 Method로 행위를 정의하는 방식<br>
URI는 명사를 사용, 행위는 GET/POST/PUT/DELETE 등으로 명확히 표현

### JWT와 세션 기반 인증 장단점

세션은 서버가 상태를 괸리해 보안성이 높지만 서버 부하가 큼<br>
JWT는 무상태성(Staeless)라 서버 확장에 유리하지만 토큰 탈취 대체 어려움

**무상태성(Stateless)** : 서버가 클라이언트의 이전 상태를 보존하지 않는 통신 특성

### WS(Nginx)와 WAS(Tomcat)의 차이
WS는 정적파일(HTML,CSS을 처리<br>
WAS는 DB조회 등 동적 로직 수행<br>
보통 Nginx를 앞단에 두어 부하 분산 후 뒷단에 Tomcat을 둠
