### Spring의 3대 핵심 요소(loC/DI , AOP, PSA)

* **loC/DI** : 제어의 역전과 의존석 주입으로 객체 간 결합도 낮춤
* **AOP**: 로깅,트랜잭션 등 공통 관심사 모듈화
* **PSA**:  서비스 추상화로 기술이 바꿔어도 코드를 유지할 수 있음

### DI(의존성 주입)가 필요한 이유와 생성자 주입을 권장하는 이유

객체간의 결합도를 낮춰 유연한 코드를 만들기 위함<br>
생성자 주입은 **불변성**을 보장하고, **순환 참조**를 컴파일 시점에서 막고 테스트 코드 작성이 용이해짐

### Spring Bean의 생명주기

스프링 컨테이너 생성 -> 빈 생성 -> 의존성 주입 -> 초기화 콜백 -> 사용 -> 소멸 전 콜백 -> 스프링 종료

### 스프링 컨테이너란?

빈의 생성, 관계 설정, 사용, 생명주기 관리 등을 관장하는 스프링의 핵심 컴포넌트

### `@Component` 와 `@Bean` 의 차이점

컴포넌트는 개발자가 직접 작성한 클래스에 붙여 빈으로 등록할 때 사용
빈은 외부 라이브러리 객체 등 개발자가 직접 제어 불가한 객체를 빈으로 등록할때 사용
