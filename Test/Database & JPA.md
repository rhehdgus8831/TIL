### JPA와 MyBatis의 차이점은?

MyBatis는 SQL을 직접 작성해 세밀한 제어가 가능하지만 반복 작업이 많음<br>
JPA는 객체와 DB를 매핑(ORM)하여 생산성이 높고 유지보수가 쉬움

### N+1 문제

1번의 쿼리로 목록을 조회했는데 연관 데이터를 가져오기 위해 N번의 추가 쿼리가 발생하는 문제<br>
`Fetch Join`으로 한 번의 쿼리로 연관 데이터까지 가져오는 걸로 해결 가능

### 즉시 로딩(Eager)과 지연 로딩(Lazy)의 차이
즉시 로딩은 연관 데이터를 한 번에 다 가져오는 것<br>
지연 로딩은 실제 사용할 때 쿼리를 날려 가져오는것<br>
불필요한 조회를 막기 위해 실무에서는 **지연 로딩** 권장

### 트랜잭션 ACID 성질
* **원자성 (Atomicity)**: 트랜잭션과 관련된 작업은 모두 성공하거나 아니면 모두 실패
* **일관성 (Consistency)**: 트랜잭션이 완료된 후에도 DB는 미리 정해진 모든 규칙을 만족해야함
* **격리성 (Isolation)**: 동시에 여러 트랜젝션이 실행될 때, 각 트랜잭션은 서러 간섭할 수 없음
* **지속성 (Durability)**: 성공적으로 완료된 트랜잭션의 결과는 시스템 장애가 발생해도 영구 기록되야 함

### DB 인덱스의 원리와 장단점은?
데이터를 빠르게 찾게 해줌<br>
조회 속도는 빠름. 데이터 삽입/수정 시 인덱스 갱신이 필요하여 성능은 느려질 수 있음

### RDBMS와 NoSQL의 차이
RDBMS는 정해진 스키마와 관계가 있어 데이터 정합성이 중요할 때 사용<br>
NoSQL은 스키마가 유연하고 확장이 쉬워 대량의 비정형 데이터 처리에 적합

### QueryDSL 사용 이유
복잡한 동적 쿼리를 자바 코드로 작성할 수 있어 컴파일 시점에 문법 오류를 잡을 수 있고 코드 가독성이 좋음
