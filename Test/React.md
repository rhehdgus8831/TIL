### React의 Virtual DOM이란 무엇이며 왜 빠른가
메모리에 가상 DOM을 뒤 변경 사항을 먼저 반영함<br>
이후 실제 DOM과 비교해 바뀐 부분만 업데이트하여 랜더링 성능이 좋음

### 컴포넌트 생명주기와 `UseEffect`는?
컴포넌트가 생성,업데이터,제거되는 과정을 말함<br>
`UseEffect` 흑 하나로 세 가지 시점의 동작을 모두 제어할 수 있음

### State와 Props의 차이점은?
`State`는 컴포넌트 내부에서 관리하는 변하는 값<br>
`Props`는 부모 컴포넌트로부터 전달받는 일기 전용 데이터

### React에서 상태 관리는 어떻게 하나?
전역 상태 관리가 필요한 경우 `Zustand`를 사용하여 간단하고 직관적으로 상태를 관리함<br>
또는 `Props`,`Children`으로 상황에 맞게 상태를 관리함

### Axois와 Fetch API의 차이는?
`Fetch`는 내장 API지만 사용이 번거로움<br>
`Axois`는 라이브러리지만 자동으로 JSON 변환을 해주며 인터셉터 기능이 강력해 실무에서 더 선호됨

### SPA{Single Page Application)의 동작 원리는?
최초에 딱 한번 페이지 전체 로딩 후 데이터만 받아와 클라이언트 측에서 화면을 부분적으로 갱신하는 방식

### 부모에서 자식으로 데이터 전달 방법
`Props`를 통해 전달

### 프론트엔드 빌드 도구
Webpack은 전통적인 번들러이며 Vite는 매우 빠른 빌드 도구임

### React Hook의 규칙
최상위 레벨에서만 호출해야함(반복문,조건문 불가)<br>
리액트 함수 컴포넌트 내에서만 호출해야함

### Props와 State의 차이점
Props는 부모 컴포넌트로부터 전달 받은 **일기 전용 데이터**<br>
State는 컴포넌트 내부에서 직접 관리하고 변경할 수 있는 **동적 데이터**

### State를 직접 수정하지않고 UseState를 사용하는 이유
React는 상태가 변경되었을 때 화면을 다시 그려야함<br>
변수를 직접 수정하면 리액트가 변경 사실을 감지하지 못해 화면이 안 바뀜<br>
그래서 상태 변경 함수를 사용해 리액트에게 알리고 리랜더링을 유도해야함

### 리액트 훅(Hooks) 중 기억나는 것 2~3가지
* `useEffect` : 부수 효과(API 호출, 타이머 등) 처리
* `useState` : 상태 관리
* `useMemo` / `useCallback` : 연산 결과나 함수를 재사용하여 성능 최적화
